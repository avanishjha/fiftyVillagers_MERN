# Fifty Villagers - Modern Education Portal Frontend

## About
This project is the modern frontend interface for the **Fifty Villagers** initiative. It is designed to provide a visually stunning, interactive, and user-friendly experience for students, parents, and the general public. The application leverages cutting-edge web technologies to deliver smooth animations, responsive layouts, and a premium aesthetic.

## Features
- **Modern Aesthetics**: Built with a focus on visual excellence, featuring glassmorphism, vibrant color palettes, and modern typography.
- **Responsive Design**: Fully optimized for all device sizes, from desktops to mobile phones.
- **Advanced Animations**: Utilizes **Framer Motion** and **GSAP** for complex, high-performance animations and transitions.
- **3D Interactions**: Integrates **React Three Fiber** for immersive 3D visual elements.
- **Component-Based Architecture**: Modular and reusable components for maintainability and scalability.

## Tech Stack
- **Framework**: [React](https://react.dev/) + [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **Animations**:
  - [Framer Motion](https://www.framer.com/motion/)
  - [GSAP](https://gsap.com/)
- **3D Graphics**: [React Three Fiber](https://docs.pmnd.rs/react-three-fiber)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Routing**: [React Router DOM](https://reactrouter.com/)

## Getting Started

### Prerequisites
Ensure you have **Node.js** installed on your machine.

### Installation
Clone the repository and install the dependencies:

```bash
git clone https://github.com/avanishjha/fiftyFrontend.git
cd fiftyFrontend
npm install
```

### Development
Start the development server with Hot Module Replacement (HMR):

```bash
npm run dev
```
The application will be available at `http://localhost:5173`.

### Production Build
Build the application for production:

```bash
npm run build
```

### Linting
Run ESLint to check for code quality issues:

```bash
npm run lint
```

## Project Structure
- `src/components`: Reusable UI components.
- `src/pages`: Page components corresponding to routes.
- `src/assets`: Static assets like images and fonts.
- `public`: Public assets served directly.
